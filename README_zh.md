# Org-GTD-Complete: 为 Emacs org-mode 设计的完整 GTD 工作流系统

[English](README.md) | [中文](README_zh.md)

本包在 Emacs org-mode 中完整实现了 David Allen 的《搞定》（GTD）方法论，提供了一套全面工作流系统。它涵盖 GTD 的五个核心步骤（收集、厘清、组织、检视、执行），并完整支持六层视野框架、自然计划模式、每周回顾流程、情境分类系统、两分钟原则以及所有标准 GTD 清单（项目、下一步行动、等待、将来/可能、参考资料）。

系统还包含收件箱处理、项目规划、定期回顾、参考资料管理、触发清单、头脑风暴工具和组织原则，帮助用户实现从日常任务到人生目标的垂直与水平关注管理。

## 安装

```elisp
M-x package-install org-gtd-complete
```

## GTD 理念

### 核心原则

GTD 方法论建立在几个根本性的哲学理念之上，这些理念构成了整个系统的基础：

1. **心智如水**：通过将任务外化到可信赖的系统中，让大脑从记忆负担中解放出来，专注于创造性思考而非存储信息。
2. **结果导向**：任何任务都必须有明确的期望结果，模糊的目标无法驱动有效行动，清晰的定义是执行的前提。
3. **系统信任**：建立一个完整、可靠的外部系统，让用户能够完全信任它，从而减少心理焦虑和决策疲劳。
4. **情境驱动**：行动的选择应基于当前的情境、可用时间、精力和工具，而非单纯的任务重要性或紧急性。
5. **持续进化**：通过定期检视和调整，系统需要不断适应变化的环境和目标，保持其相关性和有效性。

### 五个步骤

GTD 工作流包含五个连续步骤，形成一个闭环系统：

```
      +----------------+
      |      收集      |
      |    (Capture)   |
      +--------+-------+
               |
               v
      +--------+-------+
      |      处理      |
      |    (Clarify)   |
      +--------+-------+
               |
               v
      +--------+-------+
      |      组织      |
      |   (Organize)   |
      +--------+-------+
               |
               v
      +--------+-------+
      |      回顾      |
      |    (Reflect)   |
      +--------+-------+
               |
               v
      +--------+-------+
      |      执行      |
      |    (Engage)    |
      +----------------+
```

#### 1. 收集 (Capture)
将所有任务、想法、承诺和待办事项从大脑和外部来源收集到“收件箱”中。目标是清空大脑，确保没有遗漏。

#### 2. 处理 (Clarify)
对收件箱中的每个项目进行逐一处理：
- 问自己：“这是什么？”
- 如果不是可执行的任务，则丢弃、归档为参考资料或标记为“将来/可能”
- 如果是可执行任务，则确定下一步行动和期望结果

#### 3. 组织 (Organize)
将处理后的项目放入相应的系统中：
- **下一步行动清单**：可立即执行的具体行动
- **项目清单**：需要多个步骤才能完成的任务
- **等待清单**：委托他人或等待外部事件的任务
- **将来/可能清单**：暂时不执行但未来可能考虑的任务
- **参考资料**：需要保存但不需行动的信息

#### 4. 回顾 (Reflect)
定期检视系统以保持其更新和有效：
- **每日回顾**：检查当日行动清单和日程
- **每周回顾**：全面更新所有清单，回顾目标和项目进展
- **每月/年度回顾**：检视更高层次的目标和愿景

#### 5. 执行 (Engage)
基于当前情境、时间、精力和优先级，从清单中选择最合适的行动来执行。这是实际完成工作的阶段。

这五个步骤形成一个持续循环：执行后产生新的任务需要收集，定期回顾确保系统有效，从而持续提升个人生产力。

### 六层视野与双向工作流

GTD 的“六层视野”（Horizon of Focus）不仅提供了一个静态的层级框架，更重要的是它支持**双向工作流**，这是 GTD 方法论的核心精髓。

#### 两种规划路径

**1. 自上而下规划（自然计划法）**
从高层次视野出发，逐步分解为具体行动，这是实现战略目标的系统方法：
```
目的原则（L6） → 愿景（L5） → 目标（L4） → 项目（L2） → 行动（L1）
```
适用于：新项目规划、目标设定、战略思考。

**2. 自下而上处理**
从日常具体行动开始，逐步向上连接更高视野，这是应对现实工作的务实方法：
```
行动（L1） → 项目（L2） → 责任领域（L3） → 目标（L4） → 愿景（L5） → 宗旨（L6）
```
适用于：处理收件箱、日常任务执行、渐进式目标发现。

#### 动态视野演进

关键规则：
- **L4-L6 只设置在项目或职责范围层面**：高层视野不应直接关联到单个行动，而是通过项目或职责范围来体现。
- **单个行动最多关联到 L3（Area）**：具体行动可以属于某个责任领域，但不能直接拥有更高层次的视野标签。
- **行动通过所属项目继承 L4-L6**：当行动属于一个项目时，它自动继承该项目所关联的高层视野。
- **视野的渐进完善**：允许在创建行动时只设置 L1-L3，在后续回顾中逐步连接到 L4-L6，这符合认知的自然深化过程。

#### 自然计划法整合

自然计划法的五个步骤与六层视野完美对应：
1. **定义目的与原则** → L6 宗旨与核心价值
2. **展望成果** → L4-L5 目标与愿景
3. **头脑风暴** → L2-L3 项目与责任领域
4. **组织整理** → L2 项目规划
5. **明确下一步行动** → L1 具体行动

以下是六层视野与双向工作流的完整结构：

```
  +---------------------------+
  |    L6: 宗旨与核心价值     |
  |  (Purpose & Core Values)  |
  +-------------+-------------+
                ^
                |
  +----------------+----------+
  |          L5: 愿景         |
  |          (Vision)         |
  +-------------+-------------+
                ^
                |
  +-------------+-------------+
  |          L4: 目标         |
  |          (Goals)          |
  +-------------+-------------+
                ^                    ◉ 就此终止
                |                     |
  +-------------+---------------------+-----+
  |              L3: 责任领域               |
  |        (Areas of Responsibility)        |
  +-------------+---------------------+-----+
                ^                     ^
                |                     |
  +-------------+-------------+       |
  |          L2: 项目         |  不属于任何项目
  |         (Projects)        |       |
  +-------------+-------------+       |
                ^                     |
                |                     |
  +-------------+---------------------+-----+
  |             L1: 下一步行动              |
  |             (Next Actions)              |
  +-----------------------------------------+
```

关键洞察：
- L4-L6 只能通过 L2 项目继承
- 无项目的 L1 行动最高只能关联到 L3（责任领域）
- L1 始终是 L1，区别在于连接点

（注：图中使用两条不同的路径和标注来直观展示两种行动的区别：项目行动通过实线箭头向上连接到 L6，独立行动通过虚线箭头仅连接到 L3 并在该层终止。）

#### 各层详细说明

**L1: 下一步行动 (Next Actions)**
- 可立即执行的具体物理行动。
- 例如：“给客户发邮件确认会议时间”、“购买打印机墨水”。
- **关键区分**：
  - 属于项目的行动：继承该项目的 L2-L6 视野
  - 单独行动：最多关联到 L3（责任领域），不涉及 L4-L6

**L2: 项目 (Projects)**
- 需要多个步骤才能完成的期望结果。
- 每个项目应关联到至少一个责任领域（L3）。
- 例如：“完成季度报告”、“组织团队建设活动”。
- 项目本身可以设置 L4-L6 标签。

**L3: 责任领域 (Areas of Responsibility)**
- 生活中需要持续关注和维护的重要领域。
- 例如：健康、家庭、职业发展、财务管理。
- 所有行动（无论是项目内还是单独）都可以关联到责任领域。

**L4: 目标 (Goals)**
- 未来1-2年内希望实现的中期目标。
- 例如：“晋升为技术主管”、“完成马拉松训练”。
- 目标通过项目来实现，不应直接关联到单个行动。

**L5: 愿景 (Vision)**
- 未来3-5年的长期展望和生活图景。
- 例如：“成为领域专家”、“建立稳定的被动收入”。
- 愿景通过目标来具体化，进而通过项目实现。

**L6: 宗旨与核心价值 (Purpose & Core Values)**
- 人生的根本意义和指导原则。
- 例如：“帮助他人成长”、“追求科技创新”。
- 这是最高层次的视野，指导所有下层决策。

#### 实践示例：双向工作流应用

**示例1：自上而下规划（启动新项目）**
假设你的核心价值（L6）是“保持健康生活”，愿景（L5）是“拥有充沛精力”，目标（L4）是“今年完成半程马拉松”。你使用自然计划法：
1. 从 L6 核心价值出发：“保持健康生活”
2. 定义 L5 愿景：“拥有充沛精力”
3. 设定 L4 目标：“今年完成半程马拉松”
4. 创建 L2 项目：“半程马拉松训练计划”，关联到 L3 责任领域“健康”
5. 规划 L1 行动：“每周三次跑步训练”、“购买专业跑鞋”

**示例2：自下而上处理（日常任务升华）**
你在收件箱中捕获了一个想法：“今天称体重”。处理过程：
1. 识别为可执行行动（L1）
2. 判断不属于现有项目，创建为单独行动
3. 关联到责任领域（L3）“健康”
4. 在每周回顾时，系统提示：“这个行动可能支持更高目标”
5. 你将其连接到现有目标（L4）“维持健康体重”，或创建新目标
6. 行动获得更高层次的意义连接

**项目行动**：
- 项目内的下一步行动（L1）：如“每周三次跑步训练”、“购买专业跑鞋”
- 这些行动**自动继承**项目的 L2-L6 视野

**单独行动**：
- 初始状态：只关联到责任领域（L3），如“今天称体重”
- 升华潜力：通过回顾流程，可以向上连接到 L4-L6，但需要通过中间项目
- 转化路径：将单独行动升级为项目起点，或连接到现有项目的视野

在 org-gtd-complete 中，系统支持两种工作流：
1. **完整规划**：在项目层面设置 L4-L6 标签，行动自动继承
2. **渐进升华**：允许行动初始只设置 L1-L3，通过回顾流程逐步完善视野连接
3. **智能提示**：在每周回顾中检测未连接的潜力项目，引导视野升华

这种设计既保证了系统的严谨性（L4-L6 只通过项目继承），又提供了灵活性（允许视野渐进完善），真正体现了 GTD 双向工作流的精髓。

### 清单系统

GTD 的清单体系采用 **实体清单** 与 **虚拟清单** 相结合的层次结构，确保任务既能按组织结构完整存储，又能按执行状态灵活取用。

#### 关键概念区分

在理解 GTD 清单系统时，需要明确几个关键概念的区别：

**下一步行动 (Next Actions) vs 单一行动 (Single Actions)**

- **下一步行动 (Next Actions)**：指**当前可以立即执行**的具体行动。这是一个**虚拟清单**，其中的行动来自两个来源：
  1. 项目清单中每个项目的**下一个具体步骤**
  2. 单一行动清单中**当前可执行的行动**

- **单一行动 (Single Actions)**：指**不属于任何项目**的独立可执行行动。这是一个**实体清单**，存储所有不需要多步骤完成的独立任务。

**简单来说**：
- 单一行动清单是**存储**独立任务的地方
- 下一步行动清单是**动态生成**的视图，包含所有当前可执行的任务（既包括项目的下一步，也包括可执行的单一行动）

**等待清单 (Waiting For List)**：指**已委托他人或等待外部事件**的任务。这也是一个**虚拟清单**，其中的任务同样来自项目清单或单一行动清单。

#### 核心实体清单

**实体清单** 是存储任务原始记录的清单，它们是 GTD 系统的基石：

1. **项目清单 (Projects List)**
   记录所有需要多个步骤才能完成的期望结果。每个项目包含其完整计划（所有步骤）、预期成果和关联的高层视野（L4-L6）。

2. **单一行动清单 (Single Actions List)**
   记录不属于任何项目的独立可执行行动。这些行动直接与责任领域（L3）关联，不涉及项目级规划。

#### 向上映射的虚拟清单

**虚拟清单** 是根据不同视角从实体清单动态生成的视图，不重复存储任务：

- **L3 责任领域 (Areas of Responsibility)**
  映射实体清单中所有行动所属的生活领域（如健康、职业、家庭等）。

- **L4 目标 (Goals)**
  映射项目清单中为实现特定目标而设立的项目。

- **L5 愿景 (Vision)**
  映射与长期愿景相关的项目集合。

- **L6 宗旨与核心价值 (Purpose & Core Values)**
  映射反映人生核心价值与意义的所有项目。

向上映射遵循六层视野规则：**L4-L6 仅通过项目继承**；**单一行动最高映射到 L3**。

#### 向下映射的虚拟清单

基于执行状态，实体清单中的行动可动态归入以下虚拟清单：

1. **下一步行动清单 (Next Actions List)**
   包含所有 **可立即执行** 的行动，这些行动来自：
   - 项目清单中当前项目的 **下一个具体步骤**
   - 单一行动清单中 **当前可执行的行动**

2. **等待清单 (Waiting For List)**
   包含所有 **已委托他人或等待外部事件** 的行动，这些行动同样来自项目清单或单一行动清单。

向下映射遵循 **中转站原则**：当项目或单一行动中的某个步骤需要执行时，它从实体清单临时“移出”到相应的虚拟清单（下一步行动或等待清单），在实体清单中保留占位符（如“进行中”、“等待中”状态）。完成后再回归实体清单并标记为完成。

#### 辅助清单

- **收件箱 (Inbox)**：临时收集所有未处理的输入。
- **将来/可能清单 (Someday/Maybe List)**：暂不执行但未来可能考虑的想法。
- **参考资料 (Reference Material)**：不需行动但需保留的信息。

#### 实践意义

1. **完整性**：实体清单保证所有任务（项目步骤、独立行动）都有完整记录，便于整体检视。
2. **灵活性**：虚拟清单让用户只需关注当前相关视图（如“今天在办公室能做什么”、“所有等待中的项目任务”）。
3. **一致性**：任务状态变更（委托、执行、完成）仅在实体清单中记录一次，虚拟清单自动同步。

所有清单应通过 **定期回顾**（尤其是每周回顾）保持时效，确保系统持续可靠。

### 回顾流程

回顾是 GTD 系统中至关重要的环节，它确保整个系统保持最新、有效和可信。没有定期回顾，GTD 系统会逐渐失去其价值，新的任务会被遗漏，项目会失去进展，系统最终变得不可靠。

#### 回顾的核心目的

回顾过程解决几个关键问题：
- **清空大脑**：将新的任务、想法和承诺从大脑中转移到收件箱
- **更新状态**：确保所有任务的状态反映当前实际情况
- **重新校准**：根据最新情况调整优先级和时间安排
- **发现遗漏**：识别可能被忽视的项目或行动
- **保持动力**：通过看到进展来维持执行动力

#### 每日回顾

每日回顾是对当天和次日进行规划和确认的过程，建议在每天开始时进行，时长通常为10-20分钟。

**每日回顾检查清单**：

1. **查看日历**
   - 确认当天的约会和固定安排
   - 检查是否有需要提前准备的事项

2. **查看下一步行动清单**
   - 识别在当前情境下可以执行的所有行动
   - 根据情境（地点、工具、时间）筛选可用行动
   - 决定当天要执行的优先行动

3. **处理收件箱**
   - 清空自上次回顾以来的所有新输入
   - 对每个项目进行处理：是否可执行？
   - 确定下一步行动并分配到相应清单

4. **更新等待清单**
   - 检查所有等待中的项目
   - 对于逾期未完成的事项，采取跟进行动
   - 确认委托给他人的任务是否有进展

5. **查看日程安排**
   - 确认次日是否有重要安排
   - 提前准备次日需要的材料和行动

**每日回顾的最佳实践**：
- 选择一个固定的时间进行每日回顾（如每天早上工作前）
- 保持回顾时间简短但高效
- 将新想法立即写入收件箱，不要在回顾时处理

#### 每周回顾

每周回顾是 GTD 系统中最关键的回顾环节，建议在每周固定时间进行，时长通常为1-2小时。目的是全面更新所有清单，回顾项目进展，重新校准优先级。

**每周回顾检查清单**：

**第一阶段：准备与清空（15分钟）**

1. **清空收件箱**
   - 处理上周积累的所有未处理项目
   - 对每个项目进行：处理、归档或丢弃

2. **清空大脑**
   - 将所有想法、担忧、待办事项转移到收件箱
   - 打开“触发清单”（见下文）进行头脑风暴

**第二阶段：回顾与更新（45-60分钟）**

3. **回顾项目清单**
   - 逐个检查每个项目
   - 确认项目状态：进行中、已暂停、已完成
   - 识别停滞的项目，分析阻碍因素
   - 确定每个项目的下一个具体行动

4. **回顾行动清单**
   - 清理已完成或过期的行动
   - 重估优先级，调整顺序
   - 确认所有行动都有明确的下一步

5. **回顾等待清单**
   - 检查每个等待中的项目
   - 对于需要催办的事项采取行动
   - 确认委托任务的状态

6. **回顾将来/可能清单**
   - 检视暂不执行的项目
   - 是否有可以激活的项目？
   - 是否有可以丢弃的项目？

7. **回顾参考资料**
   - 整理归档的参考资料
   - 确保参考资料系统仍然有序可用

**第三阶段：展望与计划（15-30分钟）**

8. **回顾责任领域**
   - 确认各责任领域的状态
   - 识别需要关注但被忽视的领域

9. **回顾六层视野**
   - L3：是否有新的责任领域需要添加？
   - L4：当前目标是否仍然相关？是否需要调整？
   - L5：愿景是否仍然清晰？是否有新的想法？
   - L6：核心价值是否得到体现？

10. **规划下周**
    - 从项目清单中选择需要关注的项目
    - 确定下周的重点目标
    - 在日历上标记关键日期

**触发清单（用于清空大脑）**

以下是帮助清空大脑的触发问题：
- 有什么事情让你感到担忧？
- 有什么事情需要你做但你一直在拖延？
- 有什么事情需要在特定时间前完成？
- 有什么事情是你答应过别人的？
- 有什么事情是你希望完成的？
- 有什么事情可以让你的生活或工作变得更好？
- 有什么参考资料你可能需要？

#### 每月/年度回顾

月度回顾侧重于更高层次的战略思考，年度回顾则关注人生方向和长期目标。

**月度回顾检查清单**：

1. **回顾月度目标**
   - 评估上月目标的完成情况
   - 识别成功经验和改进机会

2. **回顾项目健康度**
   - 检查所有项目的整体状态
   - 识别需要重新评估或调整的项目

3. **回顾责任领域**
   - 评估各责任领域的满意度
   - 识别需要更多关注或资源的领域

4. **展望下月**
   - 设定下月的主要目标
   - 确认重要的截止日期和里程碑

**年度回顾检查清单**：

1. **评估年度目标**
   - 回顾去年设定的目标
   - 评估完成度和学习教训

2. **审视愿景**
   - 思考3-5年的生活愿景
   - 是否有需要调整的方向？

3. **审视核心价值**
   - 确认自己的核心价值是否发生变化
   - 思考如何更好地体现核心价值

4. **设定新年目标**
   - 基于愿景和价值观设定新年目标
   - 将目标分解为项目和行动

#### 回顾的节奏与习惯

建立有效的回顾习惯需要：

1. **固定时间**
   - 每日回顾：固定时间（如每天早上）
   - 每周回顾：固定时间（如周五下午或周日）
   - 每月回顾：每月第一天或最后一天
   - 年度回顾：每年一月或生日月

2. **保护时间**
   - 将回顾时间视为重要约会
   - 避免在这段时间处理其他事务

3. **建立清单**
   - 创建个人化的回顾检查清单
   - 逐步完善清单内容

4. **保持简洁**
   - 不要让回顾变得过于复杂
   - 专注于最关键的检查点

5. **持续改进**
   - 记录回顾中的问题
   - 不断优化回顾流程

通过坚持定期回顾，GTD 系统将保持活跃和有效，成为你生活和工作中真正可靠的助手。

### 情境管理

情境（Context）是 GTD 系统中用于筛选和执行行动的关键维度。情境定义了执行某个行动所需的特定条件，包括地点、工具、时间和精力等因素。通过情境管理，用户可以在特定环境下快速识别出所有可执行的行动，从而做出高效的时间分配决策。

#### 情境的核心作用

情境管理解决几个关键问题：
- **环境匹配**：根据当前所在地点和可用工具，筛选出可以立即执行的行动
- **效率优化**：避免在不适合的情境下尝试执行需要其他条件的行动
- **决策简化**：减少选择困难，在给定情境下快速做出执行决策

#### 常见情境类型

**地点情境**

地点情境根据物理位置来组织行动：
- **办公室**：需要使用电脑或在办公环境完成的任务
- **家里**：在家中可以处理的任务
- **外出**：在外出途中可以完成的任务
- **电话**：需要打电话处理的事项
- **邮件**：需要处理电子邮件的任务

**工具情境**

工具情境根据所需工具或资源来组织行动：
- **电脑**：需要使用电脑（特定软件）的任务
- **手机**：可以通过手机完成的任务
- **纸质**：需要用纸笔处理的任务
- **会议**：需要在会议中讨论的事项

**时间情境**

时间情境根据可用时间来筛选行动：
- **2分钟**：可以在2分钟内完成的快速任务
- **15分钟**：需要15分钟左右的行动
- **1小时**：需要较长时间的深度工作
- **等待时**：等待他人时的碎片时间任务

**精力情境**

精力情境根据当前精力状态来匹配行动：
- **高精力**：需要高度专注的复杂任务
- **中精力**：常规性任务
- **低精力**：简单、机械性的任务
- **创造性**：需要创意思维的任务

**社交情境**

社交情境根据是否需要与他人互动来分类：
- **独自**：可以独自完成的任务
- **会谈**：需要与他人面对面或在线交流的任务
- **团队**：需要在团队环境中处理的事项

#### 创建个人情境系统

建立情境系统需要考虑个人的生活和工作特点：

1. **分析日常活动**
   - 列出你经常出入的地点
   - 识别常用的工具和设备
   - 评估不同时段的精力状态

2. **筛选核心情境**
   - 选择最常用的5-10个情境
   - 避免情境过多导致系统复杂化
   - 定期评估情境是否仍然适用

3. **情境命名规范**
   - 使用简洁明了的名称
   - 保持命名风格一致
   - 避免歧义

#### 实践示例

假设你设定了以下情境：

**@办公室**：
- “撰写项目报告”
- “回复工作邮件”
- “整理文件归档”

**@电话**：
- “预约医生”
- “联系供应商确认交货时间”
- “给客户回电”

**@电脑**：
- “更新电子表格”
- “整理数字笔记”

**2分钟**：
- “确认收到邮件”
- “记录一个电话号码”
- “回复简单消息”

当你身在办公室时，可以快速查看“@办公室”情境下的所有行动；当你只有2分钟碎片时间时，可以从“2分钟”情境中选择一个快速任务完成。

#### 情境与项目的区别

情境和项目是两个不同的组织维度：

- **项目**关注任务的**目标**，即期望达成的结果
- **情境**关注任务的**执行条件**，即如何和在哪里执行

同一个项目下的不同步骤可能属于不同情境。例如，项目“准备季度报告”可能包含以下情境分布：
- **@办公室**：撰写报告主体
- **@电脑**：制作数据图表
- **@会谈**：向经理汇报初稿获取反馈

#### 情境管理的最佳实践

1. **情境数量适中**
   - 保持5-15个核心情境
   - 过多情境会增加维护负担
   - 过少情境会降低筛选效率

2. **情境动态更新**
   - 根据生活变化调整情境
   - 定期清理不再使用的情境
   - 保持情境与实际活动匹配

3. **与回顾结合**
   - 在每周回顾中检查情境设置
   - 评估情境是否覆盖所有行动类型
   - 优化情境结构提升执行效率

4. **灵活组合**
   - 同一行动可以关联多个情境
   - 根据实际情况选择最合适的情境视角
   - 利用情境快速切换工作模式

通过有效的情境管理，GTD 系统能够帮助用户在正确的情境下快速做出正确的事情，将有限的精力投入到最合适的行动中。

### 两分钟原则

两分钟原则是 GTD 中一个简单而有效的时间管理技巧，用于快速处理那些可以在两分钟内完成的任务。

#### 两分钟原则的核心思想

如果一个任务可以在两分钟内完成，那么立即执行它，而不是将其推迟到以后处理。这样做的好处是：
- 避免小任务堆积，减少收件箱中的项目数量
- 减少决策负担，因为不需要为小任务安排时间
- 快速完成小任务可以带来成就感，激励继续处理其他任务

#### 如何应用两分钟原则

1. **在处理收件箱时**
   - 当你处理收件箱中的项目时，如果判断该项目是可执行的，并且可以在两分钟内完成，那么立即执行它。
   - 例如：回复一封简短的邮件、确认一个会议时间、记录一个电话号码等。

2. **在执行过程中**
   - 当你在执行某个行动时，如果突然想到一个可以在两分钟内完成的小任务，可以先完成它，然后再继续原来的行动。
   - 例如：在写报告时，突然想到需要回复一封简短的邮件，可以先花两分钟回复邮件，然后再继续写报告。

3. **在回顾过程中**
   - 在每日回顾或每周回顾时，检查是否有遗漏的小任务，如果可以在两分钟内完成，就立即处理。

#### 注意事项

- **严格遵守时间限制**：只有真正能在两分钟内完成的任务才应用此原则。如果任务需要更长时间，应将其安排到下一步行动清单中。
- **避免中断**：如果两分钟的任务会打断你正在进行的深度工作，可以将其记录下来，稍后处理。
- **保持收件箱清空**：两分钟原则有助于快速清空收件箱，但不要为了追求速度而牺牲任务质量。

#### 实践示例

假设你的收件箱中有以下项目：
- “回复同事的邮件”（预计1分钟）-> 立即执行
- “给客户打电话”（预计5分钟）-> 加入下一步行动清单
- “整理桌面”（预计10分钟）-> 加入下一步行动清单
- “阅读一篇有趣的文章”（预计3分钟）-> 加入下一步行动清单

通过应用两分钟原则，你可以快速处理那些小任务，从而将更多精力集中在需要深度工作的任务上。

#### 两分钟原则的局限性

虽然两分钟原则非常实用，但它也有局限性：
- 对于复杂任务，两分钟原则不适用，需要进一步分解和规划。
- 频繁应用两分钟原则可能会导致频繁切换任务，影响深度工作的效率。
- 在某些情况下，两分钟的任务可能只是表面工作，治标不治本。

因此，两分钟原则应该与其他 GTD 原则结合使用，形成一个完整的任务管理系统。
